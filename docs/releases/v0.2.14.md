# Tube_Ultimate_Player 0.2.14

`0.2.14` 修正 Linux X11/libmpv 烟雾测试的窗口映射、加载时机和测试媒体设计，使测试更接近真实 Qt 应用生命周期。

## 上一轮诊断

`v0.2.13` 已确认：

- Qt xcb 插件成功加载
- Xvfb 24 位 X11 屏幕可用
- Mesa 软件渲染环境已启用
- libmpv 初始化成功
- `vo=gpu` 和可选 profile 配置完成

超时诊断为：

```text
position=0.0
duration=0.0
paused=false
eof=false
idle=true
```

这说明测试文件没有进入播放状态。原烟雾脚本在 Qt 事件循环启动前立即创建 libmpv 并发送 `loadfile`，顶层 QWidget 尚未完成 X11 映射；测试文件还包含音轨，而 GitHub Runner 没有普通桌面音频会话。

## 修复内容

- 对测试 QWidget 启用 `WA_NativeWindow`
- 显式获取并记录 X11 Window ID
- `show()` 后调用 `QApplication.processEvents()`，确保窗口完成映射
- 通过 `QTimer.singleShot(0, ...)` 在 Qt 事件循环中发送加载命令
- 测试媒体改为 5 秒纯视频 MP4
- 使用 FFmpeg 内置 `mpeg4` 编码器，避免额外编码器依赖
- 显式禁用音轨，排除 CI 音频设备影响

## 验证范围

- PySide6 xcb 平台初始化
- Qt 原生 X11 窗口创建与映射
- libmpv `wid` 嵌入初始化
- 本地视频加载、时长读取和播放位置推进
- 普通用户 X11/XWayland 首版技术路径
