# Tube_Ultimate_Player 0.2.17

`0.2.17` 在 `0.2.16` 已通过 Linux X11/libmpv 实际播放验证的基础上，修复 Linux 发布作业中 Qt 线程池测试可能出现的时序型段错误，确保增强 AppImage 与 DEB 构建能够稳定进入打包阶段。

## Linux 播放验证

- Linux 继续使用兼容性更高的 mpv `vo=gpu`
- 允许通过 `gpu-sw=yes` 在硬件 GPU 上下文不可用时回退到 Mesa 软件渲染
- Ubuntu 22.04 GitHub Runner 已通过 Qt xcb、X11 窗口嵌入、libmpv 初始化与播放位置推进烟雾测试
- 普通硬件环境仍优先使用 Intel、AMD 或 NVIDIA GPU

## 发布测试稳定性

Linux 发布作业会先准备捆绑的 libmpv、Deno、FFmpeg/FFprobe 与 yt-dlp，再运行完整测试。大型运行时准备改变了进程时序，暴露出测试代码中的 Qt worker 生命周期问题：

- `CreatorVideosWorker` 提交给全局 `QThreadPool` 后只保存在局部变量中
- Python 包装对象可能在线程执行期间被提前回收
- 极端时序下会导致测试进程以退出码 `139` 段错误退出

本版本在测试期间显式持有 worker，并在断言前等待线程池完成。应用本身原本已通过 `_creator_playlist_workers` 持有活动 worker，因此产品逻辑无需调整。

## 验证

- 本地完整自动化测试：`104` 项通过
- Python 编译检查通过
- `git diff --check` 通过
- Linux Test 工作流的 X11/libmpv 烟雾测试已通过

正式发布版本号更新为 `0.2.17`。
