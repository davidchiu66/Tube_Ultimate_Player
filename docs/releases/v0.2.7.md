# Tube_Ultimate_Player 0.2.7

`0.2.7` 是一次围绕播放入口、双站点 Cookie 管理、在线升级自动化与 Windows 稳定性的功能增强版本。

## 本版本重点

- 播放控制面板新增“浏览器播放”入口
- 首页和搜索结果为每个视频卡片增加独立“播放”按钮
- YouTube 与 Bilibili Cookie 内容分离，并与默认首页站点联动
- 安装包版与便携版在线升级支持确认后自动执行
- 修复播放列表动态控件的鼠标活动跟踪
- 修复 SQLite 连接未关闭造成的 Windows 数据库文件占用

## 浏览器播放

- 播放控制面板的视频信息区域新增“浏览器播放”按钮
- 点击后使用系统默认浏览器打开当前视频的原始网页链接
- 仅在当前媒体是具有有效网页地址的在线视频时启用
- 本地文件播放时自动禁用
- 浏览器无法启动或链接不可用时显示明确错误提示

## 首页与搜索结果操作

- 移除首页和搜索页顶部刷新按钮后的“播放选中”按钮
- 每个视频卡片的操作区调整为“收藏 → 下载 → 播放”
- 点击卡片中的“播放”即可进入现有 URL 解析与播放流程
- 保留双击卡片播放和单击选中反馈
- 三个操作按钮完整适配现有卡片宽度

## 双站点 Cookie 管理

- YouTube 与 Bilibili Cookie 内容分别保存到独立文件
- 设置页的 Cookie 编辑框与“默认首页”选择联动：
  - 选择 Bilibili 时显示 Bilibili Cookie
  - 选择 YouTube 时显示 YouTube Cookie
- 在两个站点之间切换时，尚未保存的编辑内容会分别保留
- 视频解析、首页、搜索、播放列表和 Bilibili API 请求会根据实际目标 URL 选择对应 Cookie
- 旧版单 Cookie 配置会按升级时选择的默认首页自动归属，避免原有内容丢失

## 自动在线升级

升级包下载完成后不再只提示用户手动处理，而是先请求用户确认。

### 安装包版

- 用户确认后安排关闭当前应用
- 外部启动器等待当前进程完全退出
- 文件锁释放后自动启动新版安装程序
- 启动失败时保留日志并显示错误提示

### 便携版

- 用户确认后启动独立便携版升级进程
- 等待当前应用退出后自动解压 ZIP
- 使用可靠的文件复制流程覆盖程序目录
- 替换完成后自动启动新版应用
- 开发源码模式禁止自动覆盖工作区

## 播放列表活动跟踪

- 播放列表条目属于运行时动态创建的控件，原先没有完整安装播放器鼠标事件过滤器
- 现在列表刷新后会为新条目及其子控件安装鼠标跟踪
- 鼠标在标题、封面或条目内部移动时会重置播放器空闲计时器
- 播放列表面板和播放控制器不会在用户操作期间意外隐藏

## SQLite 稳定性修复

- 修复将 `sqlite3.Connection` 直接用于 `with` 时只提交事务但不关闭连接的问题
- 新增统一的事务连接上下文，退出时保证提交/回滚并关闭连接
- 收藏、历史和播放列表仓储全部改用新的连接生命周期
- 解决 Windows 下临时数据库仍被占用、完整测试随机失败的问题
