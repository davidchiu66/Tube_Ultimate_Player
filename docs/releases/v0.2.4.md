# Tube_Ultimate_Player 0.2.4

`0.2.4` 是一次以性能整改和交互补强为主的维护版本，重点完成了首页/搜索结果缓存、缩略图缓存、下载持久化优化，以及播放器工具栏和停止回跳逻辑的增强。

## 本版本重点

- 完成 P0 阶段性能整改，减少重复网络请求、重复图片加载与高频磁盘写入
- 新增首页/搜索页结果缓存，显著改善重复进入、翻页返回时的响应速度
- 新增进程内缩略图缓存，并合并相同缩略图的并发请求
- 优化下载任务 JSON 持久化与下载目录扫描，降低下载过程中的磁盘压力
- 顶部工具栏新增“置顶”按钮，支持一键切换窗口置顶/普通窗口
- 调整播放器停止逻辑，停止后自动返回视频的来源页面

## 性能优化

### 首页与搜索缓存

- 为 YouTube / Bilibili 首页结果增加页级内存缓存
- 为搜索结果增加关键字 + 页码级别缓存
- 刷新按钮保持真实刷新语义，不走缓存
- 设置保存、Cookie 变化、代理变化、默认首页变化后，缓存会自然失效

### 缩略图缓存

- 新增共享缩略图缓存服务 `ui/thumbnail_cache.py`
- 首页九宫格与播放列表面板共用同一套缩略图缓存
- 相同 URL 和尺寸的缩略图只下载一次
- 相同请求会合并等待，避免重复解码与重复缩放

### 首页布局重排优化

- 首页卡片重排增加 debounce
- 仅在列数发生变化时才重新布局
- 改善窗口拖拽、最大化、尺寸频繁变动时的 UI 稳定性

### 下载流程优化

- 下载进度更新改为节流写入 `download_tasks.json`
- 任务新增、删除、完成、失败仍立即保存
- 持久化改为临时文件写入后原子替换，降低任务文件损坏风险
- 下载进度文件匹配改为缓存已识别文件路径，避免每秒全目录扫描

## 交互增强

### 工具栏置顶按钮

- 顶部工具栏新增图钉样式“置顶”按钮
- 点击后窗口切换为最上层显示
- 再次点击后恢复普通窗口
- 全屏和普通窗口模式下都可正确切换状态

### 停止后返回来源页

- 如果当前播放视频来自首页，点击“停止”后返回首页
- 如果当前播放视频来自搜索页，点击“停止”后返回搜索结果页
- 如果当前播放视频来自下载列表，点击“停止”后返回下载列表
- 同时兼容收藏、历史、播放列表等其它来源入口，遵循“从哪里来，回哪里去”
