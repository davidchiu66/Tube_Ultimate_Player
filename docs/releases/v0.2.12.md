# Tube_Ultimate_Player 0.2.12

`0.2.12` 修复 Ubuntu 22.04 系统 libmpv 初始化时不支持 `profile=fast` 而导致播放器启动中止的问题。

## 问题定位

`v0.2.11` 已成功完成以下 Linux 验证：

- Ubuntu 依赖安装
- 全部 Linux 单元测试
- PySide6 xcb 平台插件加载
- Xvfb 真实 X11 窗口创建
- 系统 libmpv 动态库加载

播放器随后在设置初始化选项时失败：

```text
MpvError: 设置 mpv 选项失败: profile: error setting option
```

Ubuntu 22.04 的 mpv/libmpv 版本不保证提供名为 `fast` 的预设 profile。该 profile 只是性能调优项，不是窗口嵌入、解码或播放所需的核心配置。

## 修复内容

- 保留 `profile=fast` 作为可选优化
- 支持该 profile 的 libmpv 继续启用
- 不支持时记录警告并继续初始化
- `wid`、`vo`、硬件解码、缓存等关键选项仍保持强校验
- 新增单元测试，模拟 libmpv 拒绝 `profile` 并确认初始化不会中止

## Linux 首版能力

- Ubuntu 22.04/24.04 x86_64
- X11 与 Wayland 会话中的 XWayland/xcb
- AppImage 捆绑 libmpv
- 增强 AppImage/DEB 自带 Deno、FFmpeg/FFprobe 和 yt-dlp
- XDG 目录和 Linux 浏览器 Cookie Profile
- root 安装支持与 root 运行警告
- Linux 更新包手动安装策略
- Windows/Linux 联合发布门禁
