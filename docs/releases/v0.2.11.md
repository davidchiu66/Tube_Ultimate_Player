# Tube_Ultimate_Player 0.2.11

`0.2.11` 补齐 Ubuntu 上 PySide6 xcb 平台插件所需的系统运行库，使 Xvfb/X11 环境可以进入真实 libmpv 嵌入播放验证。

## 问题现象

`v0.2.10` 已修复烟雾测试的项目模块导入路径，Linux 单元测试全部通过。X11 烟雾测试随后进入 Qt 初始化阶段，但 Qt 输出：

```text
Could not load the Qt platform plugin "xcb" in "" even though it was found.
```

Qt 同时提示 `xcb-cursor0`，但该提示是 xcb 插件初始化失败时的通用说明。工作流已经安装 `libxcb-cursor0`，实际还需要补齐 PySide6 6.11 使用的其他 xcb 辅助库。

## 修复内容

Ubuntu 测试和发布构建现在显式安装：

- `libx11-xcb1`
- `libxcb1`
- `libxcb-cursor0`
- `libxcb-icccm4`
- `libxcb-image0`
- `libxcb-keysyms1`
- `libxcb-randr0`
- `libxcb-render-util0`
- `libxcb-shape0`
- `libxcb-shm0`
- `libxcb-sync1`
- `libxcb-xfixes0`
- `libxcb-xinerama0`
- `libxcb-xkb1`
- `libxkbcommon-x11-0`

烟雾测试同时启用 `QT_DEBUG_PLUGINS=1`。如果仍有动态库缺失，远端日志会直接显示具体 `.so` 名称，而不是只显示 Qt 的通用 xcb 错误。

## 继承的 Linux 功能

- Ubuntu 22.04/24.04 x86_64
- X11 与 Wayland 会话中的 XWayland/xcb
- 捆绑 libmpv 的增强 AppImage
- 自带 Deno、FFmpeg/FFprobe、yt-dlp 的增强 AppImage/DEB
- XDG 目录、Linux 浏览器 Cookie Profile、root 使用警告
- Linux 升级包下载和手动安装策略
- Windows 与 Linux 构建共同作为正式发布门禁
