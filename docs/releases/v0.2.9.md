# Tube_Ultimate_Player 0.2.9

`0.2.9` 启动 Linux 桌面版本交付，提供 Ubuntu x86_64 的 X11/XWayland 运行支持、增强 AppImage/DEB 构建链路，并完善跨平台目录、Cookie、运行时和升级策略。

## 本版本重点

- 新增 Ubuntu 22.04/24.04 x86_64 支持
- Wayland 桌面通过 XWayland/xcb 运行
- Linux AppImage 必须捆绑 libmpv
- 优先构建自带 Deno、FFmpeg/FFprobe、yt-dlp 的增强 AppImage 与 DEB
- 新增 Linux 浏览器 Cookie Profile 探测
- 新增 XDG 运行时目录及 root 使用边界
- 新增 Ubuntu Xvfb/libmpv 自动播放烟雾测试
- 正式发布工作流同时构建 Windows 与 Linux 产物

## Linux 播放器与显示系统

- 应用启动前检测 Linux 桌面会话
- Wayland 会话在未显式设置 Qt 平台时自动选择 `QT_QPA_PLATFORM=xcb`
- 原生 X11 继续使用 Qt `winId()` 与 libmpv `wid` 嵌入
- 若用户强制使用原生 Wayland，播放器会给出明确的暂不支持提示
- 新增 `TUBE_PLAYER_QPA_PLATFORM` 高级覆盖变量
- 启动日志记录系统平台、桌面会话、Qt 后端和 root 状态

首版不包含 libmpv Render API 重构，因此不承诺原生 Wayland 嵌入播放。

## libmpv 与增强运行时

- Linux libmpv 加载支持包内 `libmpv.so.2`、`libmpv.so.1`、`libmpv.so` 和系统 `find_library`
- AppImage 从包内 `3rdpart/` 优先加载 libmpv
- Linux yt-dlp 查找支持 `yt-dlp_linux` 与 `yt-dlp`
- 自动优先使用包内 `deno`、`ffmpeg` 和 `ffprobe`
- Linux 不再误用 Windows Node MSI 或 FFmpeg 7z 自动安装流程
- 缺少运行时时显示发行版安装或增强版使用说明

## XDG 目录

Linux 运行数据按用途拆分：

```text
$XDG_CONFIG_HOME/Tube_Ultimate_Player/
$XDG_DATA_HOME/Tube_Ultimate_Player/
$XDG_CACHE_HOME/Tube_Ultimate_Player/
$XDG_STATE_HOME/Tube_Ultimate_Player/logs/
$XDG_VIDEOS_DIR/Tube_Ultimate_Player/
```

- 配置和站点 Cookie 写入 XDG config
- SQLite 与下载任务写入 XDG data
- 缩略图、Cookie 转换和升级包写入 XDG cache
- 日志写入 XDG state
- 支持从 `user-dirs.dirs` 读取用户视频目录
- 忽略不符合 XDG 规范的相对路径变量

## Linux 浏览器 Cookie

新增检测：

- Google Chrome
- Chromium
- Brave
- Vivaldi
- Firefox
- Chromium/Firefox Snap
- Chrome/Chromium/Brave/Firefox Flatpak

浏览器列表仍只显示实际存在 Cookie 数据库的 Profile。非标准安装目录使用绝对 Profile 路径交给 yt-dlp；Firefox 手动读取也支持绝对 Profile 目录。Linux Cookie 文件创建后限制为 `0600`。

## root 安装与运行

- 支持通过 root/sudo 安装 DEB
- 不要求 AppImage 使用 root
- 不硬性禁止 root 启动应用
- root 启动时显示安全和兼容性警告
- `/root` 配置、桌面密钥环、浏览器 Cookie、PipeWire/PulseAudio 和显示会话可能与当前桌面用户隔离
- root 创建的下载文件归 root 所有，应用不会自动放宽权限或修改所有权

## Linux 在线升级

- 自动识别增强 AppImage 和 DEB Release 资产
- 优先选择带 `_with_deno_ffmpeg` 后缀的增强资产
- 下载完成后打开所在目录并提供手动安装说明
- 不调用 `sudo`、`pkexec`、`apt`、`dpkg` 或 `dnf`
- 不自动替换 AppImage，也不自动安装 DEB

## AppImage 与 DEB 构建

新增：

- `build_linux.py`
- `packaging/linux/prepare_runtime.sh`
- AppRun 与 DEB 启动器
- `.desktop` 桌面入口
- `test-linux.yml`
- `release-linux.yml`

增强构建会：

1. 准备 Deno、FFmpeg/FFprobe、yt-dlp 和系统 libmpv
2. 保存许可证、版本、FFmpeg 构建配置和源代码地址
3. 使用 PyInstaller 创建 Linux one-dir 应用
4. 使用 linuxdeploy 收集 libmpv 用户态动态依赖闭包
5. 生成增强 AppImage 和 DEB
6. 生成下载文件、第三方运行时和最终共享库 SHA256 清单
7. 校验桌面文件和包内容

禁止捆绑 glibc、GPU 驱动和主机 Mesa 驱动。
