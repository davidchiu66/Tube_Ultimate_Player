# Tube_Ultimate_Player 0.2.15

`0.2.15` 为 Linux X11/libmpv 实证增加原生播放器日志和加载时序诊断，用于准确定位媒体保持 idle 状态的底层原因。

## 当前验证状态

Ubuntu Runner 已确认：

- Qt xcb 插件加载成功
- Xvfb 原生 X11 窗口已映射，并获得有效 Window ID
- Mesa 软件渲染环境可用
- 系统 libmpv 创建并初始化成功
- `wid`、`vo=gpu` 和兼容选项配置成功

测试媒体仍保持：

```text
duration=0
position=0
idle=true
```

仅通过属性无法判断加载命令是否进入 libmpv、解复用失败、VO 初始化失败，或者文件被立即卸载。

## 新增诊断

- 新增环境变量 `TUBE_PLAYER_MPV_LOG_FILE`
- 设置该变量时，在 libmpv 初始化前配置 `log-file`
- 同时请求 `msg-level=all=v` 详细日志
- 不设置环境变量时完全保持原有日志行为
- `log-file` 和 `msg-level` 作为可选选项，不影响旧版 libmpv 启动
- 烟雾测试打印加载命令发送前后的 idle 状态
- 超时时输出 libmpv 日志最后 120 行

该日志可能包含本地媒体路径，因此只在自动化测试或用户明确诊断时启用。

## 验证

- 本地完整自动化测试：`104` 项通过
- 直接脚本导入检查：通过
- X11 窗口映射：远端已通过
- libmpv 初始化：远端已通过
- 媒体加载失败原因：由 Ubuntu Runner 日志继续确认

正式发布版本号更新为 `0.2.15`。
