# Tube_Ultimate_Player 0.2.5

`0.2.5` 是一次围绕 DLNA 投屏、播放工作流和数据列表体验的功能收口版本。

## 本版本重点

- 新增完整的 DLNA 发送端能力，支持在线视频和本地媒体投屏
- 新增作者作品动态播放列表，单视频播放后在后台加载，不阻塞当前播放
- 完善停止、全屏、自然结束重播和播放列表切换时的状态恢复
- 下载、收藏和播放历史统一增加来源信息与本地搜索
- 收藏、历史、下载列表统一为黑底金边表格风格
- 版本号更新为 `0.2.5`，补齐正式发布说明和发布流程校验

## DLNA 投屏

- 支持 SSDP 多网卡设备发现，并解析 MediaRenderer、AVTransport、RenderingControl 控制地址
- 支持远程播放、暂停、停止、进度同步、Seek 和音量控制
- 支持在线视频单流中继、分离音视频 FFmpeg 实时封装
- 支持下载列表中的本地视频和音频，通过临时局域网 HTTP 服务投屏
- 首次扫描结果在当前会话内缓存；后续打开投屏窗口时并行校验设备 IP + 端口
- 缓存设备全部失效时自动回退完整 SSDP 扫描，手动刷新始终执行完整扫描
- 投屏失败、切换媒体或应用退出时清理临时媒体服务和 FFmpeg 进程

## 播放器与播放列表

- 单视频播放后后台生成作者作品动态播放列表，避免影响首帧播放
- 明确播放列表、作者动态列表和本地保存列表保持各自上下文，不互相覆盖
- 播放器自然播放结束后支持从头重播
- 停止后按实际来源返回首页、搜索、收藏、历史、下载或播放列表页面
- 修复从首页播放视频后切换作者播放列表条目，点击停止时返回来源丢失的问题
- 修复最大化窗口进入视全屏后停止或退出无法恢复最大化状态的问题
- 优化控制面板鼠标移出后的隐藏和全屏交互

## 下载、收藏与历史

- 下载任务保存 YouTube / Bilibili 来源信息，并兼容旧 JSON 任务记录
- 收藏和播放历史数据库新增来源字段，历史记录新增作者字段
- 旧 SQLite 数据库启动时自动补充字段，并根据历史 URL 修正 Bilibili 来源
- 下载列表、收藏和播放历史增加标题/来源/作者/链接等本地搜索筛选
- 收藏和历史改为统一表格布局，增加来源、作者、时间和行内操作
- 下载、收藏、历史列表统一使用黑底金边主题，改善深色主题下的可读性

## 配置与兼容性

- 新增 DLNA 媒体服务端口配置，默认端口为 `8899`
- 新增 DLNA 设备发现超时配置
- 保留现有 YouTube、Bilibili、Cookie、代理、FFmpeg、JS Runtime 和下载目录配置
- 运行时数据继续写入 `%LocalAppData%\Tube_Ultimate_Player`
