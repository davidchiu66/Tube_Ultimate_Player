# Tube_Ultimate_Player 0.2.1

`0.2.1` 是首个按正式版本流程发布的稳定版本，重点完善了版本发布链路，并继续补强播放器桌面化体验。

## 本版本重点

- 顶部工具栏升级为更现代的深色播放器风格，补齐图标、搜索区与统一按钮样式
- “关于”按钮调整为直接进入关于页面，关于页继续承担版本检测、升级入口与发布说明展示
- 版本检测兼容正式版与预发布版发布记录，修复仅存在预发布版本时返回 HTTP 404 的问题
- 发布工作流改为正式版本发布方式，不再按 `prerelease` 发布
- GitHub Actions 在构建阶段自动补齐 `libmpv-2.dll`、`libEGL.dll`、`libGLESv2.dll`，修复便携版/安装包打包失败

## 功能与体验

### 播放与浏览

- 支持 URL 解析播放 YouTube 视频
- 支持 YouTube 首页推荐与搜索分页
- 支持九宫格卡片展示、双击播放、全屏切换、单击播放/暂停
- 支持清晰度、字幕、音量、倍速等控制

### 收藏、历史与下载

- 支持收藏、历史记录与再次播放
- 支持下载队列、暂停、继续、删除、本地播放
- 支持配置下载目录、并发下载数与 FFmpeg 目录

### 设置与环境

- 支持直接粘贴 Cookie 文本并持久化
- 支持自动探测系统浏览器 Cookie 来源
- 代理优先级为：系统代理优先，其次手动配置代理
- 支持 JS Runtime 检测与 Node.js 安装引导

### 版本与升级

- 新增关于页面中的版本信息、Release Note、检测更新与在线升级入口
- 升级包下载目录统一位于 `%LocalAppData%\\Tube_Ultimate_Player\\updates`

## 发布与工程

- 运行期配置、日志、数据库、下载目录统一位于 `%LocalAppData%\\Tube_Ultimate_Player`
- 仓库不再提交 `3rdpart` 二进制，发布时由工作流动态下载依赖
- 便携版与安装包统一发布到 GitHub Releases

## 说明

- 本版本开始，后续版本发布统一按正式版本流程执行
- 在线升级当前仍以“下载升级包并提示后续操作”为主，尚未引入独立热更新替换器
