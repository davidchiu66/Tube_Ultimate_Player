# Tube_Ultimate_Player 0.2.16

`0.2.16` 允许 Linux libmpv 在硬件 GPU 上下文不可用时回退到 Mesa 软件渲染，解决 CI、虚拟机和部分远程桌面环境无法初始化视频输出的问题。

## libmpv 日志结论

`v0.2.15` 的详细日志确认：

- MP4 文件成功打开
- libavformat 正确识别容器
- MPEG-4 视频轨成功选择
- X11 display `:99` 成功打开
- EGL 1.5 与 OpenGL 4.5 上下文成功创建
- Mesa llvmpipe 软件渲染器成功识别

随后 mpv 因默认策略拒绝疑似软件渲染器：

```text
Detected suspected software renderer.
Suspected software renderer or indirect context.
Failed initializing any suitable GPU context!
Error opening/initializing the selected video_out (--vo) device.
```

这不是窗口嵌入、媒体解析或 OpenGL 缺失，而是 mpv 对软件 GPU 的默认保护策略。

## 修复内容

- Linux 增加 `gpu-sw=yes`
- 该选项允许软件 GPU 作为兜底，但不会强制普通用户使用软件渲染
- 有可用 Intel/AMD/NVIDIA 硬件上下文时仍优先硬件 GPU
- Xvfb、虚拟机和远程桌面可使用 Mesa llvmpipe
- `gpu-sw` 作为可选兼容项，旧版 libmpv 不支持时只记录警告
- Windows 保持原有 `gpu-next` 路径，不额外启用软件 GPU 选项

## Linux 首版技术路径

- Qt xcb/XWayland
- libmpv `wid` 嵌入
- `vo=gpu`
- 硬件 GPU 优先，Mesa 软件 GPU 兜底
- AppImage 捆绑 libmpv
- 增强 AppImage/DEB 捆绑 Deno、FFmpeg/FFprobe 和 yt-dlp
