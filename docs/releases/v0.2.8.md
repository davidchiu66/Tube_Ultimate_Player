# Tube_Ultimate_Player 0.2.8

`0.2.8` 重点改进播放器暂停体验、浏览器 Cookie 来源展示，并新增自带 Deno 与 FFmpeg 的增强安装包和免安装便携版。

## 本版本重点

- 暂停状态下控制面板也能按空闲时间自动滑出
- Cookie 浏览器列表仅显示系统实际检测到的可用浏览器/Profile
- 新增自带 Deno、FFmpeg、FFprobe 的增强安装包
- 新增自带运行时的免安装增强便携版
- 正式发布流程同时生成四种 Windows 产物
- 下载任务按实际视频站点选择对应 Cookie

## 播放器控制面板

- 暂停状态不再强制让播放控制面板永久显示
- 暂停与播放现在共用相同的鼠标热区、空闲计时和滑入滑出逻辑
- 暂停后控制面板先保持可见，空闲约 3.2 秒后自动滑出
- 鼠标移动到底部热区或操作播放列表时重新滑入并重置计时
- 播放结束状态仍保持控制面板常驻，方便执行重播或停止操作

该调整避免暂停画面中的字幕、文字或其他重要内容长期被底部控制区域遮挡。

## 浏览器 Cookie 来源

- 设置页不再人为列出 Chrome、Firefox、Brave、Opera 等全部浏览器名称
- 仅显示系统实际检测到、并存在可读取 Cookie Profile 的浏览器
- 保留“自动检测”和“不从浏览器读取”两种通用模式
- 旧配置若指向已卸载或已无法检测的浏览器，会自动回到“自动检测”

## 自带 Deno 与 FFmpeg 的增强安装包

新增可独立运行、也可由正式发布流程调用的增强构建工作流：

- 自动下载官方 Deno Windows x64 版本
- 自动下载 FFmpeg essentials Windows 构建
- 集成 `deno.exe`、`ffmpeg.exe` 和 `ffprobe.exe`
- 集成相关第三方许可文件
- 输出文件名：
  `Tube_Ultimate_Player_setup_v0.2.8_with_deno_ffmpeg.exe`

安装后应用会优先检测并使用内置 Deno；现有 FFmpeg 检测逻辑会直接使用内置 FFmpeg，不再提示用户额外下载安装。

## 自带运行时的免安装便携版

新增独立构建脚本：

```bash
python build_portable_with_deno_ffmpeg.py
```

也可以使用：

```bash
python build_portable.py --with-deno-ffmpeg
```

输出文件名：

```text
Tube_Ultimate_Player_portable_v0.2.8_with_deno_ffmpeg.zip
```

压缩包包含应用、Deno、FFmpeg、FFprobe、README 和第三方许可声明，解压后即可运行。

## 发布工作流

正式 `release.yml` 现在会构建并发布四种产物：

1. 普通便携版 ZIP
2. 自带 Deno/FFmpeg 的增强便携版 ZIP
3. 普通 Windows 安装包 EXE
4. 自带 Deno/FFmpeg 的增强安装包 EXE

增强构建也可以通过 `release-installer-with-deno-ffmpeg.yml` 单独手动运行。

## Cookie 下载修复

- 下载任务不再根据默认首页选择 Cookie

- 跨站点下载时能够正确选择各自独立保存的 Cookie 文件
