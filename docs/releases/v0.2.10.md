# Tube_Ultimate_Player 0.2.10

`0.2.10` 延续 `0.2.9` 的 Linux 首版适配，并修复首次 Ubuntu CI 实证中发现的 X11/libmpv 烟雾测试启动路径问题。

## 本版本重点

- 提供 Ubuntu 22.04/24.04 x86_64 的 X11/XWayland 支持
- 提供自带 libmpv、Deno、FFmpeg/FFprobe、yt-dlp 的增强 AppImage 和 DEB 构建链路
- Linux 使用 XDG 配置、数据、缓存、状态和视频目录
- 支持 Linux 原生、Snap、Flatpak 浏览器 Cookie Profile 探测
- Linux 更新只下载资产，不自动提权安装
- 修复 Ubuntu Runner 直接执行播放烟雾测试时无法导入项目模块的问题

## CI 修复

首次运行 `test-linux.yml` 时，依赖安装和全部单元测试均通过，但下面的命令以脚本文件方式执行：

```bash
xvfb-run -a env QT_QPA_PLATFORM=xcb python tests/linux_mpv_smoke.py
```

Python 此时默认只把 `tests/` 加入模块搜索路径，导致：

```text
ModuleNotFoundError: No module named 'player'
```

烟雾测试现在会根据自身路径计算仓库根目录，并在导入应用模块前将其加入 `sys.path`。该修复不会改变正式应用的模块加载方式。

## Linux 技术范围

- 首期架构：`x86_64/amd64`
- 正式支持：X11、Wayland 会话中的 XWayland/xcb
- 暂不支持：原生 Wayland libmpv 嵌入、ARM64、RPM、Flatpak/Snap 应用包
- AppImage 必须捆绑 libmpv
- 增强包优先捆绑 Deno、FFmpeg/FFprobe 和 yt-dlp
- DEB 可通过 root/sudo 安装，正式运行建议使用普通桌面用户
