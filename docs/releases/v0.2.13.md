# Tube_Ultimate_Player 0.2.13

`0.2.13` 改进 Ubuntu 22.04/Xvfb 环境下的 libmpv 图形输出兼容性，并增强远端播放烟雾测试诊断。

## 实证进展

上一轮 Ubuntu Runner 已验证：

- PySide6 xcb 平台插件加载成功
- Xvfb 真实 X11 窗口创建成功
- 系统 libmpv 创建和初始化成功
- 可选 `profile=fast` 不再阻断启动

但短视频加载后，8 秒内没有观察到播放位置推进。旧版 mpv 在无物理 GPU 的 Xvfb 环境下使用 `vo=gpu-next` 可能无法获得稳定的渲染上下文。

## 修复内容

- Linux 默认视频输出由 `gpu-next` 调整为成熟且兼容范围更广的 `gpu`
- Windows 继续使用 `gpu-next`，不改变现有播放路径
- Xvfb 显式创建 `1280x720x24` 屏幕
- CI 设置 `LIBGL_ALWAYS_SOFTWARE=1`，强制使用 Mesa 软件渲染
- 烟雾测试超时从 8 秒增加到 12 秒
- 超时时输出位置、时长、暂停、EOF、idle 状态和测试文件大小
- 新增 Linux/Windows 默认视频输出选择测试

## 支持边界

- Linux x86_64
- X11 与 Wayland 会话中的 XWayland/xcb
- Ubuntu 22.04/24.04
- AppImage 捆绑 libmpv
- 增强 AppImage/DEB 捆绑 Deno、FFmpeg/FFprobe 和 yt-dlp
- 原生 Wayland Render API 仍不属于首版范围

## 验证

- 本地完整自动化测试：`104` 项通过
- Linux Qt/xcb 初始化：已通过
- Linux libmpv 初始化：已通过
- Linux X11 实际播放位置推进：由 Ubuntu Runner 重新验证

正式发布版本号更新为 `0.2.13`。
