# 视频下载功能需求备案

备案日期：2026-07-06

## 1. 背景

当前播放器已经支持通过 yt-dlp 解析 YouTube 视频并交给 libmpv 播放。新增“视频下载”功能后，用户可以在播放当前视频时将其加入下载队列，并在独立的“下载列表”页面查看下载进度、管理队列、播放已下载文件。

本功能应复用现有的代理、Cookie、JS Runtime 配置，避免出现“能播放但不能下载”的配置分裂。

## 2. 功能范围

### 2.1 设置项

在设置页面新增以下配置：

1. 视频保存路径
   - 默认值：当前应用目录下的 `downloads`。
   - 应用启动或保存设置时，如果目录不存在，应自动创建。
   - 用户可以通过“浏览”按钮选择新的保存目录。
   - 下载完成的视频保存到该目录。

2. 同时下载视频数
   - 默认值：`1`。
   - 允许范围：`1` 到 `10`。
   - 保存时需要做边界限制，小于 1 按 1，大于 10 按 10。
   - 并发数变化后，下载管理器应按新值调度后续任务。

3. FFmpeg 目录
   - 默认值为空。
   - 用户可以指定 `ffmpeg.exe` 所在目录，也可以指定 `ffmpeg.exe` 文件路径。
   - 下载命令需要通过 `--ffmpeg-location` 传给 yt-dlp。
   - FFmpeg 检测顺序：用户配置目录、`3rdpart` 目录、系统 PATH。
   - 检测到 FFmpeg 时，优先下载当前清晰度的视频流和最佳音频流并合并；未检测到时，降级为 yt-dlp 的最佳单文件格式。

### 2.2 播放器下载入口

在视频播放控制面板增加“下载”按钮。

按钮行为：

1. 当前没有已解析视频时禁用。
2. 当前视频可播放后启用。
3. 点击后，将当前视频加入下载队列。
4. 如果当前下载中的任务数量小于设置的并发下载数，则立即开始下载。
5. 如果已经存在相同 URL 的任务：
   - 等待中、下载中、暂停：不重复添加，只提示任务已存在。
   - 已完成：提示已完成下载，并可在下载列表中双击播放。
   - 失败：允许用户通过下载列表“启动”重新下载。

### 2.3 顶部导航入口

在顶部工具栏增加“下载列表”按钮，放置在“播放器”按钮后面。

导航顺序建议：

```text
播放 URL | 首页 | 播放器 | 下载列表 | 历史 | 设置
```

点击“下载列表”进入下载列表页面。

### 2.4 下载列表页面

下载列表展示当前和历史下载任务。

建议列：

1. 标题
2. 清晰度
3. 状态
4. 进度
5. 速度
6. 剩余时间
7. 保存路径
8. 操作

任务状态：

```text
等待中
下载中
暂停
已完成
失败
已删除
```

“已删除”状态可以不在列表中长期展示，第一版可直接从列表移除。

### 2.5 队列操作

下载队列需要支持“暂停、启动、删除”三类操作。

#### 暂停

适用状态：

```text
等待中
下载中
```

行为：

1. 等待中的任务：状态改为“暂停”，不参与并发调度。
2. 下载中的任务：终止当前 yt-dlp 子进程，状态改为“暂停”。
3. 暂停时保留已下载的临时文件、`.part` 文件和任务记录。
4. 暂停后立即释放一个并发下载槽，并尝试调度下一个等待任务。

说明：

yt-dlp 本身没有通用的“暂停进程但保留网络连接”能力，因此暂停实现为“停止当前下载进程并保留断点文件”。再次启动时，依赖 yt-dlp 的断点续传能力继续下载。

#### 启动

适用状态：

```text
暂停
失败
等待中
```

行为：

1. 暂停任务：改为“等待中”，由下载管理器按并发数调度。
2. 失败任务：清理错误信息，改为“等待中”，允许重新下载。
3. 等待任务：如果并发下载槽可用，则立即开始；否则保持等待。
4. 启动下载时应使用 yt-dlp 的断点续传能力，继续使用已存在的 `.part` 文件。

#### 删除

适用状态：

```text
等待中
暂停
失败
已完成
下载中
```

行为：

1. 等待、暂停、失败任务：从队列列表中移除。
2. 下载中任务：先终止 yt-dlp 子进程，再从队列列表中移除。
3. 已完成任务：从下载列表中移除，但默认不删除本地已下载文件。
4. 第一版删除操作不主动删除本地视频文件和 `.part` 文件，避免误删用户数据。
5. 后续可扩展“删除任务并删除本地文件”的二次确认操作。

### 2.6 下载完成提示

当任务完成下载后：

1. 在下载列表中标注“已完成”。
2. 记录本地文件路径。
3. 在主窗口右下角弹出 toast 提示。

Toast 文案建议：

```text
下载完成：<视频标题>
```

Toast 显示 3 秒左右自动消失，不打断当前播放。

### 2.7 已完成视频播放

在下载列表中双击已完成任务：

1. 如果本地文件存在，则调用现有 libmpv 播放该文件。
2. 播放本地文件不再调用 yt-dlp 解析。
3. 如果本地文件不存在，则将任务标记为“文件丢失”或提示“文件不存在”。

## 3. 下载格式策略

默认策略：

1. 优先下载当前播放器正在使用的清晰度。
2. 如果当前清晰度是独立视频流，则同时下载最佳音频流。
3. 如果当前清晰度信息不可用，则使用 yt-dlp 默认最佳策略：

```text
bestvideo+bestaudio/best
```

建议命令格式：

```text
yt-dlp.exe
  --newline
  --continue
  --merge-output-format mp4
  --progress-template "%(progress._percent_str)s|%(progress._speed_str)s|%(progress._eta_str)s"
  -o "<save_dir>/%(title).200B [%(id)s].%(ext)s"
  -f "<format_selector>"
  <url>
```

命令需要继续复用：

1. 系统代理优先，其次配置代理。
2. Cookie 文件或浏览器 Cookie。
3. JS Runtime。
4. socket timeout、retries、fragment retries 等现有稳定性参数。

## 4. 设计方案

### 4.1 新增模块

建议新增：

```text
download/
  __init__.py
  models.py
  download_manager.py
  download_worker.py
  command_builder.py

ui/
  download_page.py
  toast.py
```

也可以将 `download` 放入 `services` 或 `workers`，但独立包更清晰，后续便于扩展任务持久化、重试、限速等能力。

### 4.2 数据模型

核心字段：

```text
task_id
url
video_id
title
quality_label
format_selector
status
progress
speed_text
eta_text
save_dir
output_path
error_message
created_at
updated_at
```

### 4.3 DownloadManager

职责：

1. 管理下载队列。
2. 控制并发下载数量。
3. 接收 worker 进度、完成、失败信号。
4. 暴露暂停、启动、删除接口。
5. 下载完成后发出 toast 通知信号。
6. 设置保存后刷新并发限制。

### 4.4 DownloadWorker

职责：

1. 构造并启动 yt-dlp 子进程。
2. 按行读取 stdout/stderr。
3. 解析下载进度并发出进度信号。
4. 支持外部取消/暂停时终止子进程。
5. 完成后识别最终文件路径。

### 4.5 DownloadPage

职责：

1. 展示下载任务列表。
2. 展示进度条、速度、剩余时间、状态。
3. 提供暂停、启动、删除按钮。
4. 双击已完成任务播放本地文件。

按钮状态建议：

```text
等待中：暂停可用，启动不可用，删除可用
下载中：暂停可用，启动不可用，删除可用
暂停：暂停不可用，启动可用，删除可用
失败：暂停不可用，启动可用，删除可用
已完成：暂停不可用，启动不可用，删除可用
```

### 4.6 MainWindow 集成

新增连接：

1. `PlayerPage.download_requested -> MainWindow._download_current_video`
2. `DownloadPage.play_file_requested -> MainWindow.play_local_file`
3. `DownloadManager.task_changed -> DownloadPage.update_task`
4. `DownloadManager.task_completed -> Toast.show_message`

新增导航：

```text
download_nav = QPushButton("下载列表")
```

加入 `QStackedWidget`：

```text
DownloadPage
```

### 4.7 本地文件播放

新增 `MainWindow.play_local_file(path)`：

1. 切换到播放器页面。
2. 调用 `self.mpv.load(path)`。
3. 设置播放器按钮为可用。
4. 更新标题为本地文件名。

## 5. 日志要求

新增日志分类：

```text
tube_player.download
```

需要记录：

1. 任务入队。
2. 任务开始。
3. 暂停、启动、删除操作。
4. 下载完成路径。
5. 下载失败错误。
6. yt-dlp 下载命令，敏感信息需要脱敏。

## 6. 验收标准

1. 设置页可以配置下载目录和并发下载数。
2. 默认下载目录为应用目录下 `downloads`。
3. 播放视频后，“下载”按钮可用。
4. 点击“下载”后任务进入下载列表。
5. 并发下载数量不超过设置值。
6. 下载中能看到进度、速度、剩余时间。
7. 暂停下载后任务状态变为“暂停”，且不继续占用并发槽。
8. 启动暂停任务后可以继续下载。
9. 删除任务后列表不再显示该任务。
10. 下载完成后右下角出现 toast 提示。
11. 下载完成任务在列表中显示“已完成”。
12. 双击已完成任务可以直接用 libmpv 播放本地文件。
13. 播放本地文件不触发 yt-dlp 解析。
14. 代理、Cookie、JS Runtime 与播放解析保持同一套配置。

## 7. 暂不纳入第一版的能力

以下能力可后续扩展：

1. 下载速度限制。
2. 下载字幕。
3. 下载封面。
4. 下载任务持久化到 SQLite。
5. 删除任务时同时删除本地文件。
6. 下载完成后自动打开所在目录。
7. 右键菜单操作。
8. 批量下载首页或播放列表。
