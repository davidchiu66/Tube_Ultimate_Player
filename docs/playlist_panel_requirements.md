# Tube_Ultimate_Player 播放列表详情页与侧滑面板实施备案

更新日期：2026-07-09

## 1. 目标

本轮实现的目标，是让“列表类内容”从一次性解析播放，升级为完整的播放列表工作流：

1. 支持列表详情页
2. 支持播放器上的右侧滑入播放列表面板
3. 支持命名保存与加载
4. 支持自动连播
5. 支持批量下载

适用对象包括：

- YouTube playlist / mix / album 类结果
- Bilibili 多 P、番剧、收藏夹、合集、稍后再看等列表型内容
- 用户手动保存的播放列表快照

## 2. 已实现能力

### 2.1 列表详情页

新增独立列表详情页，支持：

1. 展示列表标题、来源、封面、条目数量
2. 展示视频条目列表
3. 双击条目开始播放
4. 返回播放器页
5. 保存当前列表
6. 批量下载条目

### 2.2 播放器右侧面板

播放器中新增右侧播放列表面板，支持：

1. 固定宽度
2. YouTube 风格的封面 + 两行文字布局
3. 当前播放项高亮
4. 双击条目切换播放
5. 顶部下拉框切换已保存列表
6. 自动连播开关
7. 批量下载入口
8. 取消按钮关闭面板

### 2.3 滑入滑出交互

当前交互规则：

1. 仅在存在已保存播放列表时，进入播放器页面才允许触发面板滑入
2. 鼠标进入右侧热点区域时，面板从右向左滑入
3. 以下情况会从左向右滑出：
   - 双击条目开始播放
   - 鼠标离开面板与热点区域
   - 鼠标空闲 3.2 秒

### 2.4 选择与批量操作

支持：

1. 单击选中
2. `Ctrl` 多选
3. 下载选中
4. 全部下载

命名与描述上的最终口径：

- 原“全选”按钮已调整为“取消”按钮，用于关闭面板

## 3. 数据模型与持久化

新增仓储：

- `database/playlist_repository.py`

持久化内容包括：

1. 命名播放列表基础信息
2. 播放列表条目快照
3. 自动连播属性
4. 来源 URL 与来源站点

设计原则：

1. 保存的是“条目快照”，不是仅保存源 URL
2. 这样即使上游动态列表内容变化，本地保存列表仍然稳定可回放

## 4. 页面与状态协同

### 4.1 主窗口上下文

主窗口维护当前有效播放列表上下文，包括：

1. 当前列表 key
2. 当前列表条目
3. 当前播放索引
4. 自动连播状态

### 4.2 播放流程

列表类内容的播放链路：

1. 先解析出列表条目
2. 打开列表详情页或直接装入播放器面板上下文
3. 用户双击某一条目
4. 再对该单条视频执行真实解析
5. 播放成功后更新当前索引与高亮状态

### 4.3 自动连播

当满足以下条件时触发自动连播：

1. 当前存在有效播放列表
2. 自动连播已开启
3. 当前项不是最后一项
4. 播放自然结束

## 5. 验证方法

### 5.1 列表详情页

1. 输入 YouTube playlist URL 或 Bilibili 列表 URL
2. 确认进入列表详情页
3. 双击任意条目，确认进入播放器并正常播放
4. 点击返回，确认返回播放器页而不是首页

### 5.2 侧滑面板

1. 保证存在至少一个已保存播放列表
2. 进入播放器页面
3. 鼠标移动到右侧热点区，确认面板滑入
4. 鼠标移出或空闲 3.2 秒，确认面板滑出
5. 双击条目播放后，确认当前项高亮更新且面板关闭

### 5.3 选中与批量下载

1. 单击条目，确认可见选中高亮
2. `Ctrl` + 单击多个条目，确认多选有效
3. 执行批量下载，确认任务进入下载队列

### 5.4 持久化

1. 保存一个命名播放列表
2. 重启应用
3. 在面板下拉框中重新加载
4. 确认条目、顺序与自动连播配置仍存在

## 6. 风险与后续建议

1. 超长列表当前仍以可用性优先，后续可考虑懒加载与更强的虚拟化
2. 列表封面与标题来自上游数据，个别来源可能缺字段，需要继续做兜底
3. 若未来支持跨站点混合自建播放列表，可以在当前仓储模型上继续扩展

## 7. 结论

播放列表能力已从“仅支持单视频播放”升级到“可保存、可加载、可连播、可批量下载”的完整第一版，可作为后续专辑、合集、多站点队列管理的基础能力继续演进。
