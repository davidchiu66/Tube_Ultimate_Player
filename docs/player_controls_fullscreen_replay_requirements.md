# Tube_player 播放控制、全屏恢复与结束重播优化方案

状态：方案已审核通过，编码与非联网验证已完成  
备案日期：2026-07-15

## 1. 背景与目标

本轮聚焦播放器的三个交互问题：

1. 播放控制面板目前只能依靠 3.2 秒空闲超时滑出，鼠标主动离开控制区域后反馈不够及时。
2. 主窗口在最大化状态进入视全屏后，点击“停止”会回到普通窗口，而不是恢复到进入视全屏前的最大化状态。
3. 视频自然播放结束后，“播放”按钮仍沿用普通暂停/继续逻辑，不能可靠地从头重新播放。

目标是在不改变现有解析、清晰度切换、字幕、播放列表自动连播和停止返回页面规则的前提下，完善上述状态转换。

## 2. 现状分析与原因

### 2.1 控制面板只有超时滑出

当前 `PlayerPage` 的鼠标移动处理只包含以下分支：

- 鼠标命中控制面板或底部 72px 热区时，调用 `_show_controls()`。
- 每次鼠标活动都重启 3.2 秒 `_idle_timer`。
- 只有 `_handle_idle_timeout()` 会把 `_controls_visible` 设为 `False` 并启动滑出动画。

缺少“鼠标从控制面板有效区域移动到外部”的状态转换，因此主动移出后只能等待超时。

### 2.2 停止后丢失最大化状态

当前 `_toggle_fullscreen()` 退出视全屏时无条件调用 `showNormal()`；`_stop_playback()` 在视全屏中停止时也无条件调用 `showNormal()`。

`showNormal()` 的语义是恢复普通窗口，而不是恢复“进入视全屏前的窗口状态”。代码又没有保存进入视全屏前是否为最大化，所以“最大化 -> 视全屏 -> 停止”最终必然变为普通窗口。

### 2.3 播放结束后不能重新播放

当前播放按钮直接连接 `MpvPlayer.toggle_pause()`。自然播放结束时，mpv 处于 EOF（`eof-reached`）状态；EOF 不是普通暂停，仅切换 `pause` 属性不会可靠地将播放位置重置到 0 秒。

现有 `playback_finished` 信号只用于播放列表自动连播，没有把播放器 UI 和播放按钮切换为明确的“已结束，可重播”状态。

## 3. 完善后的需求

### 3.1 控制面板滑出规则

控制面板有效区域定义为以下区域的并集：

1. 控制面板当前可见区域。
2. 播放器底部 72px 的唤出热区。

仅当视频正在播放、未加载、未暂停时启用自动隐藏，行为如下：

1. 鼠标进入有效区域，控制面板滑入或保持显示。
2. 鼠标从有效区域移到视频区等外部区域，控制面板立即启动 220ms 滑出动画，不再等待 3.2 秒。
3. 鼠标在滑出动画期间重新进入底部热区，动画反向切换为滑入。
4. 鼠标空闲 3.2 秒仍作为兜底：控制面板滑出，同时隐藏光标。
5. 因“鼠标移出”触发滑出时只隐藏控制面板，不立即隐藏光标；光标仍由空闲超时控制。
6. 暂停、加载、停止或无媒体时，控制面板保持可见。

交互保护：

- 鼠标仍位于控制面板任一子控件时，不得误判为离开。
- 拖动进度条、音量条期间不得因鼠标短暂越界而隐藏；释放鼠标后再按当前位置判断。
- 清晰度、字幕、倍速下拉框的弹出列表打开时不得隐藏；弹出列表关闭后再按当前鼠标位置判断。
- 播放列表右侧面板维持现有独立热点和隐藏策略，不纳入本次控制面板区域判断。

### 3.2 视全屏退出后的窗口状态恢复

进入视全屏前记录主窗口状态，第一版需要区分：

- 普通窗口。
- 最大化窗口。

所有退出视全屏的路径使用同一个恢复方法：

1. 点击“退出全屏”。
2. 双击视频退出视全屏。
3. 视全屏中点击“停止”。

恢复规则：

- 从最大化进入视全屏，退出或停止后调用 `showMaximized()`。
- 从普通窗口进入视全屏，退出或停止后调用 `showNormal()`，并保持原普通窗口尺寸和位置。
- 恢复后显示顶部工具栏，播放器页边距和“全屏”按钮文案恢复正常。
- 视全屏期间切换置顶状态不得覆盖进入视全屏前的窗口状态快照。
- 停止后的页面返回逻辑保持现状，只修正主窗口状态。

### 3.3 播放结束后的重新播放

引入明确的自然播放结束状态，区分“播放中、暂停、已结束”。

1. 单个视频或播放列表最后一项自然结束后，按钮显示“播放”，控制面板保持可见。
2. 在“已结束”状态点击“播放”，将当前媒体定位到 0 秒并继续播放。
3. 普通暂停后点击“播放”仍从当前位置继续，不回到 0 秒。
4. 重新播放成功后清除“已结束”状态，进度和按钮恢复播放中状态。
5. 加载新视频、切换播放列表条目、停止播放时清除旧的“已结束”状态。
6. 播放列表开启自动连播且存在下一项时，继续按现有逻辑播放下一项，不短暂进入可重播状态。
7. 播放列表关闭自动连播或已经到最后一项时，允许重播当前项。
8. 本地文件和当前支持的在线视频均采用相同行为。

## 4. 设计方案

### 4.1 `ui/player_page.py`

补充控制面板热点进入/离开状态跟踪：

- 保存上一次鼠标是否位于控制面板有效区域。
- 在统一鼠标移动处理中计算坐标并识别“区域内 -> 区域外”的转换。
- 离开时调用独立的 `_hide_controls()`，复用现有位移动画。
- 保留 `_handle_idle_timeout()`，但将“隐藏面板”和“隐藏光标”职责拆开，避免主动移出时同步隐藏光标。
- 在拖动和活动弹出框期间抑制离开隐藏，交互结束后重新判断。

补充播放结束 UI 状态入口，例如 `set_playback_finished(finished: bool)`，确保结束后按钮显示“播放”、控制面板可见，并在新媒体开始时复位。

### 4.2 `ui/main_window.py`

增加进入视全屏前的状态快照，例如 `_was_maximized_before_fullscreen`，并集中封装：

- `_enter_player_fullscreen()`：只在真正进入时记录状态、隐藏工具栏并调用 `showFullScreen()`。
- `_leave_player_fullscreen()`：按快照调用 `showMaximized()` 或 `showNormal()`，恢复工具栏和播放器布局。

`_toggle_fullscreen()` 与 `_stop_playback()` 统一调用上述方法，避免不同退出路径行为漂移。

播放按钮不再直接绑定 `mpv.toggle_pause`，改由主窗口处理：

- 普通状态：切换暂停/继续。
- 已结束状态：调用 mpv 重播能力，从 0 秒开始并清除结束状态。

`_handle_playback_finished()` 先判断是否满足播放列表自动连播；有下一项则保持现有连播流程，否则设置自然结束状态。

### 4.3 `player/mpv_player.py`

增加明确的重播方法，例如 `restart()`：

1. 对可定位媒体执行 `seek 0 absolute+exact`。
2. 清除内部 EOF 边沿检测状态。
3. 设置 `pause=no` 继续播放。
4. 对不可定位媒体保留最近一次 `load()` 参数并重新加载，作为兜底。
5. 命令失败时沿用现有 `MpvError` 错误处理链路，不静默吞错。

不修改 `keep-open=yes`，继续保留视频结束后的最后一帧，以免扩大行为变更范围。

### 6.2 控制面板手工验证

1. 普通窗口播放视频，鼠标移入底部热区，确认面板滑入。
2. 鼠标从面板向上移入视频区，确认面板立即开始滑出，光标仍可见。
3. 面板滑出过程中移回底部热区，确认面板重新滑入且无闪烁。
4. 鼠标停在视频区 3.2 秒，确认面板和光标均隐藏。
5. 分别操作进度条、音量条、倍速、清晰度、字幕下拉框，确认操作期间面板不误隐藏。
6. 暂停视频并移出热点，确认面板保持显示。
7. 在视全屏下重复以上步骤，行为应与普通窗口一致。

### 6.4 播放结束与回归验证

1. 播放一个在线视频至自然结束，点击“播放”，确认从 00:00 重新开始。
2. 播放一个本地文件至自然结束，重复上述验证。
3. 播放中途暂停后点击“播放”，确认从暂停位置继续，而不是从头播放。
4. 自然结束后拖动或重播，确认进度条、时间文本和按钮状态同步。
5. 播放列表开启自动连播，当前项结束后确认自动进入下一项。
6. 播放列表最后一项结束后点击“播放”，确认重播最后一项。
7. 播放列表关闭自动连播，当前项结束后点击“播放”，确认重播当前项。
8. 播放结束后点击“停止”，确认原有停止、返回页面和按钮禁用逻辑不受影响。
