# 搜索、URL 弹窗、播放控制器自动隐藏与收藏功能需求备案

更新时间：2026-07-07

## 1. 背景

当前应用已经支持：

- 首页加载 YouTube 推荐视频并以卡片展示。
- 输入 URL 后解析并播放。
- 播放页通过 libmpv 播放视频，并支持暂停、停止、清晰度、字幕、全屏、下载。
- 历史记录持久化到 SQLite。
- 下载列表已具备持久化和本地播放能力。

本次增强目标是在不破坏现有播放、下载、历史功能的前提下，引入搜索入口、URL 播放弹窗、统一的控制器自动隐藏体验，以及收藏视频管理能力。

## 2. 需求整理

### 2.1 搜索入口与搜索结果

1. 顶部工具栏在“播放URL”按钮前增加“搜索”按钮。
2. 当前顶部 URL 输入框改为搜索关键词输入框。
3. 用户输入关键词后点击“搜索”，调用 yt-dlp 搜索 YouTube 视频。
4. 搜索结果复用首页九宫格卡片形式展示。
5. 搜索结果需要支持分页。
6. 首页推荐视频能力继续保留。
7. 搜索结果卡片支持：
   - 单击选中。
   - 双击播放。
   - 点击“播放选中”播放。
   - 后续增加收藏入口。

### 2.2 播放 URL 弹窗

1. 顶部“播放URL”按钮不再直接读取顶部输入框。
2. 点击“播放URL”后弹出对话框。
3. 对话框包含：
   - URL 输入框。
   - “播放”按钮。
   - “取消”按钮。
4. 点击“播放”后校验 URL，并调用现有解析播放流程。
5. 点击“取消”关闭对话框，不改变当前页面和播放状态。

### 2.3 播放控制器自动隐藏策略

当前体验：

- 全屏状态下鼠标空闲 3.2 秒后隐藏控制面板和光标。
- 鼠标移动后控制面板重新显示。

新需求：

1. 窗口模式和全屏模式都启用同一套空闲检测。
2. 播放状态下，鼠标空闲 3.2 秒后：
   - 播放控制器面板滑出隐藏。
   - 鼠标光标隐藏。
3. 鼠标移动时：
   - 光标立即显示。
   - 如果鼠标位于播放控制器面板区域，控制器滑入显示。
   - 如果鼠标在视频区域或其他非控制器区域移动，控制器不滑入。
4. 鼠标空闲再次超过 3.2 秒后，光标再次隐藏。
5. 非播放状态或加载状态下，控制器默认保持可见，避免用户找不到操作入口。

需要补充约定：

- “播放控制器面板区域”建议定义为播放器底部控制面板当前可见区域，隐藏后保留一个底部热区，例如播放器底部 72px。
- 鼠标进入底部热区时视为进入控制器区域，触发滑入。
- 退出底部热区后不立即隐藏，仍由 3.2 秒空闲计时器负责隐藏，避免面板闪烁。

### 2.4 收藏视频功能

1. 顶部“历史”按钮前增加“收藏”按钮。
2. 新增收藏页面，用于管理收藏视频。
3. 首页卡片增加“收藏”按钮或收藏图标。
4. 播放控制界面增加“收藏”按钮。
5. 收藏信息需要持久化。
6. 收藏页面支持：
   - 展示收藏视频列表。
   - 双击播放。
   - 删除收藏。
   - 搜索/过滤收藏，建议作为第一版可选项，如实现成本低则一并做。
7. 重复收藏同一视频时不新增重复记录，只更新收藏时间或提示已收藏。

## 3. 设计方案

### 3.1 搜索服务

优先使用 yt-dlp 的搜索能力，避免引入 YouTube Data API Key。

建议命令形式：

```text
yt-dlp --dump-single-json --flat-playlist --playlist-end <page_size> "ytsearch<page_size>:<keyword>"
```

分页方案：

1. 第一版使用“页码 + 每页数量”的轻量方案。
2. 每页固定 45 条。
3. 第 N 页使用 `ytsearch{N * page_size}:keyword` 获取前 N 页结果，然后截取最后一页。
4. 优点：实现简单，与现有 yt-dlp 调用风格一致。
5. 缺点：页码越大越慢。第一版可接受，后续再优化为 YouTube Data API 或更细粒度搜索源。

新增模块建议：

- `workers/search_worker.py`
- `resolver.youtube_resolver.search_videos(keyword, page, page_size)`

搜索结果模型：

- 可复用 `HomeVideo`。
- `HomePage` 可扩展为支持 `mode=home/search`，避免新增重复的 SearchPage。

### 3.2 首页/搜索结果页面复用

复用当前 `HomePage`：

- 增加状态字段：
  - `mode`: `home` / `search`
  - `keyword`
  - `page`
  - `page_size`
- 增加分页按钮：
  - “上一页”
  - “下一页”
  - 页码显示
- 首页模式下分页控件隐藏。
- 搜索模式下分页控件显示。

顶部搜索按钮行为：

1. 读取顶部搜索框内容。
2. 内容为空则提示。
3. 切换到首页页面。
4. 设置首页为搜索加载状态。
5. 启动 `SearchWorker`。
6. 成功后调用 `HomePage.set_videos(...)`，并设置搜索状态标题，例如“搜索：关键词，第 1 页”。

### 3.3 播放 URL 弹窗

新增轻量对话框：

- 可以放在 `ui/main_window.py` 内部方法实现，也可以新增 `ui/url_dialog.py`。
- 推荐新增 `ui/url_dialog.py`，避免主窗口继续膨胀。

接口：

```python
class UrlPlayDialog(QDialog):
    def url(self) -> str: ...
```

主窗口：

- `play_url_button.clicked -> _show_play_url_dialog`
- 对话框接受后调用现有 `play_url(url)`。
- 顶部搜索框回车建议触发搜索，不再触发播放 URL。

### 3.4 控制器自动隐藏

建议将控制器显示逻辑尽量下沉到 `PlayerPage`，主窗口只负责全屏切换。

`PlayerPage` 增加：

- `set_auto_hide_enabled(enabled: bool)`
- `notify_mouse_activity(pos: QPoint)`
- `_is_in_control_hot_zone(pos: QPoint) -> bool`
- `_hide_cursor_and_controls()`
- `_show_cursor_only()`
- `_show_controls()`

行为：

1. 播放开始后启用自动隐藏。
2. 播放暂停、停止、加载时关闭自动隐藏或保持控制器可见。
3. 鼠标移动：
   - 总是显示光标。
   - 重置 3.2 秒空闲计时器。
   - 如果命中控制器区域或底部热区，显示控制器。
   - 否则只显示光标，不显示控制器。
4. 计时器超时：
   - 隐藏光标。
   - 控制器滑出。

窗口模式和全屏模式复用同一逻辑：

- 控制面板仍保持 overlay 形式。
- 窗口模式下也允许滑出到底部。
- 全屏模式下隐藏顶部工具栏逻辑保持不变。

### 3.5 收藏数据持久化

沿用当前 SQLite 结构和 `HistoryRepository` 的模式。

新增表：

```sql
CREATE TABLE IF NOT EXISTS favorite (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id TEXT NOT NULL UNIQUE,
    title TEXT NOT NULL,
    webpage_url TEXT NOT NULL,
    uploader TEXT,
    duration INTEGER DEFAULT 0,
    thumbnail TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

新增模块建议：

- `database/favorite_repository.py`
- `ui/favorite_page.py`

收藏入口：

1. 首页卡片：
   - 增加“收藏”按钮或星标按钮。
   - 点击后调用 `favorite_repository.add(video)`。
2. 播放控制器：
   - 增加“收藏”按钮。
   - 当前 `current_video` 存在时可用。
3. 收藏页：
   - 列表或卡片展示收藏视频。
   - 双击播放。
   - 删除收藏。

第一版建议使用列表页，稳定后再改成卡片网格，原因是收藏管理更偏向检索和删除，列表操作更清晰。

## 4. UI 调整方案

顶部工具栏目标顺序：

```text
[搜索关键词输入框........................] [搜索] [播放URL] [首页] [播放器] [下载列表] [收藏] [历史] [设置]
```

搜索框 placeholder：

```text
搜索 YouTube 视频
```

播放 URL 弹窗：

```text
播放 URL

[请输入 YouTube URL........................]

                         [取消] [播放]
```

首页/搜索结果：

- 固定正方形卡片继续保留。
- 搜索模式显示分页栏。
- 首页模式不显示分页栏。
- 每页 45 条。

播放器控制器：

```text
[播放] [停止] [下载] [收藏] 音量 倍速 清晰度 字幕 [全屏]
```

## 5. 风险与边界

1. yt-dlp 搜索分页不是真正的 YouTube 游标分页，页码越大越慢。
2. 搜索结果字段可能比首页推荐少，部分视频可能缺少 duration/uploader/thumbnail。
3. 从 yt-dlp 搜索结果播放时，仍需要二次解析具体视频 URL，这是正常流程。
4. 控制器“只在底部热区滑入”的体验需要仔细调试，否则用户可能误以为控制器丢失。
5. 收藏按钮需要避免重复点击导致 UI 状态和数据库状态不一致。

## 6. 实施步骤

建议分 5 个小步实施，每步可独立验证：

1. 搜索入口与搜索 Worker
   - 顶部输入框改搜索语义。
   - 增加搜索按钮。
   - 新增搜索 worker 和 resolver 搜索方法。
   - 搜索结果复用 HomePage。

2. 搜索分页
   - HomePage 增加上一页/下一页。
   - 增加搜索状态保存。
   - 页大小固定 45。

3. 播放 URL 弹窗
   - 新增 UrlPlayDialog。
   - 播放 URL 按钮改为打开弹窗。
   - 回车行为改成搜索。

4. 播放控制器自动隐藏重构
   - 窗口/全屏统一空闲检测。
   - 鼠标移动只显示光标。
   - 底部热区触发控制器滑入。

5. 收藏功能
   - SQLite 表和 repository。
   - 收藏页。
   - 首页收藏按钮。
   - 播放控制器收藏按钮。

## 7. 验证方法

### 7.1 搜索

1. 输入关键词 `python qt mpv`，点击“搜索”。
2. 首页区域显示 45 条搜索结果。
3. 点击“下一页”，结果刷新，页码变化。
4. 点击“上一页”，返回上一页。
5. 双击任一搜索结果，可以进入播放器并播放。
6. 搜索加载期间，刷新/播放选中按钮状态正确。

### 7.2 播放 URL 弹窗

1. 点击“播放URL”，出现弹窗。
2. 输入 `https://www.youtube.com/watch?v=6WTwnW95yQ4`，点击播放，进入解析播放。
3. 点击取消，弹窗关闭，不改变当前页面。
4. 输入空 URL 或非法 URL，有明确提示。

### 7.3 控制器自动隐藏

1. 窗口模式播放视频，鼠标不动 3.2 秒，控制器滑出，光标隐藏。
2. 窗口模式移动鼠标到视频中部，光标出现，控制器不滑入。
3. 窗口模式移动到底部控制器热区，控制器滑入。
4. 全屏模式重复上述三步。
5. 暂停时控制器保持可见。
6. 停止播放后回到首页，自动隐藏不再干扰首页。

### 7.4 收藏

1. 首页点击收藏按钮，toast 提示收藏成功。
2. 同一视频重复收藏，不产生重复记录。
3. 播放页点击收藏，当前视频进入收藏列表。
4. 点击顶部“收藏”，进入收藏页。
5. 收藏页双击视频，可以播放。
6. 收藏页删除视频后，该记录消失。
7. 重启应用后，收藏记录仍存在。

## 8. 第一版验收标准

1. 顶部搜索和播放 URL 两条路径互不干扰。
2. 搜索结果能稳定展示、分页、播放。
3. 播放控制器在窗口和全屏下自动隐藏行为一致。
4. 收藏可新增、去重、展示、删除、重启后保留。
5. 所有新增失败路径均写入日志。
6. 不破坏现有首页、播放、下载、历史、设置功能。
## 9. 2026-07-07 补充备案

1. 由于 `yt-dlp` 搜索和分页搜索在网络较慢或 Cookie 校验较重时等待时间会明显长于首页推荐，搜索流程必须显示显式的等待文案。
2. 第一版采用不确定进度条加温馨提示语，而不是伪造百分比，避免误导用户。
3. 建议文案示例：
   - `正在搜索 YouTube，请稍等，这一步通常会比首页加载更慢一些...`
   - `正在搜索 YouTube：<关键词>（第 N 页），请稍等...`
4. 搜索等待期间，首页区域中的“刷新”“播放选中”“上一页”“下一页”应根据状态自动禁用，防止重复触发后台任务。
